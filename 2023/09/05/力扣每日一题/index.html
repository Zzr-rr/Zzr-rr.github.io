2337. 移动片段得到字符串

解题思路

​ 类似于这种判断第一个单词是否能用第二个单词变换后得到的题目，首先应该判断两个字符串的长度是否相等，不相等的字符串直接返回false即可。

​ target如果能够由start移动后得到，那么target中的所有’L’一定比start中的‘L’更加’L’，target中的所有’R’一定比start中的’R’更加’R’。

​ 也就是说，start中的所有第i个L一定在target中的第i个L的左边，并且L和R的顺序不能变。

代码思路

    1. 使用`vector<pair<char, int>>`的数据结构，分别存储两组数据，用来记录方向和坐标。
    1. 依次将`start`数组和`target`数组存储之后，开始遍历先前存储的两组数据。
    3. 遍历过程中需要注意以下几点
   - L和R的总个数相等，即两组数据的长度相等
   - L和R的出现顺序相等
   - L和R的位置

849. 到最近的人的最大距离

解题思路

夹在两人之间的最大距离为这两人距离的1/2，且向上取整，就是说如示例一，0和4，距离为3，那么他们之间的最大距离为(4 - 0)//2 = 2。需要单独考虑的是边界问题，也就是如果是[0,0,0,1,...]的情况，那么它的距离应该为0的个数。

最后，在遍历的过程中不断更新最大值，就可以得到相应的最大距离了。

关于如何推导出来两个人之间的距离为(j - i) // 2，我们可以这样思考：

两个中间的空位，要么是奇数，要么是偶数，空位的数量为j - i - 1

  • 奇数情况下，空位数(j - i - 1) // 2 + 1就是答案。
  • 偶数情况下，和比它小1的奇数空位是相同的，也就是说(j - i - 1 - 1) // 2 + 1 = (j - i) // 2就是答案。
  • 考虑到奇数情况下的(j - i - 1) % 2 = 1，因此，将余数的1减去对答案没有影响，可以统一为(j - i - 1 -1) // 2 + 1 = (j - i) // 2 。

综上，两个人之间的距离就是(j - i) // 2.

1267. 统计参与通信的服务器

解题思路

方法一：

顺序遍历每一台服务器，查看是否能与其他服务器通讯，如果能够通讯，就将服务器的台数加一。

遍历采取如下方式：

  • 大框架为按行按列，从上往下，从左往右的顺序遍历每一个点
  • 遍历到1，则判断是否有服务器能与之通讯，因为我们遍历的顺序为从上往下，从左往右，这里的判断是本题的关键，一般情况下，可以遍历该点所在行和所在列的每一个点，但是注意到我们的遍历顺序，可以发现如果使用从该点开始往右、往下找的规则去寻找，则它左上方的服务器一定遍历过它本身。
  • 因此，按照往右、往下找的顺序，找到1，则将它记录为2，标志着该服务器已与其它服务器连接通讯，按照此方法，则可以减少一半的搜索量。

方法二：

记录每一行每一列的服务器数量，再进行遍历，如果该行或者该列的服务器的数量大于等于2，则可通讯的服务器的数量加一。

使用两个数组分别记录列和行的服务器数量。

解题代码

代码细节：

​ 在定义dfs函数时，sum_变量传入的是引用，而max_变量传入的是形参，这是因为：

  1. sum_变量需要在函数外定义，并且作为全局参数不断更新，如果仅仅传入的形参，那么sum_会被拷贝一份，而不会在原来的值上进行修改。
  2. max_只需要在当前的函数中判断大小即可，作为一个判断条件，可以仅仅是拷贝的一个值。如果作为引用传入，那么在遍历左子树如果更新了最大值，在遍历右子树时，这个最大值就不是所在路径的最大值了，而是它的兄弟节点的值，会导致错误。

57. 插入区间

解题思路

​ 这道题目就是一句话，在intervals中找到和newInterval有交集的区间的情况合并为一个大区间，如果没有有交集的情况，就将他插入到最后或者最前面。

​ 这道题目思路比较清晰，但是实现起来有很多的细节需要注意。

  • 首先是如何判断区间是否相交，对于原来的某个区间，它要和newInterval交集的话，则它满足以下两个条件中的一个：

    1. 该区间的右端点在newInterval中 。

    2. 该区间的左端点在newInterval中。

  • 其次我们需要对没有交集的区间进行处理，分为两种情况，interval的右端点在newInterval的左端点的左侧，interval的左端点在newInterval的右端点的右侧，对于这种情况，直接在ans中加入该interval。

  • 换种思路，如果interval和newInterval不属于第二点的两种情况，那么他们应该是有交集的，可以进一步简化代码

    • 对于有交集的情况，如何合并为大区间？首先我们需要明确的是，有交集的情况下，该有交集的区间一定是连续的（因为原来的代码是按照顺序排列的），因此我们可以不断更新该大区间的左端点和右端点。
    • 首先记录newInterval的左端点为left，右端点为right,不断更新left为最小值和right为最大值。
    • 更新结束，向ans中添加{left, right}，继续遍历在newInterval右侧的区间。

  • 对于特殊情况，就是在遍历完整个intervals之后，如果newInterval还没有被加入到数组中去，那么需要在最后将其加入到ans中去。

  • 最后返回ans

823. 带因子的二叉树

解题思路

​ 构造这样一颗二叉树，它一定是有左右孩子的，我们不妨先看看构造一棵二叉树的过程，首先确定一个根节点，其次在数组中找到它的因子，不妨设其中一组分别为它的左孩子和右孩子，再从它的左孩子出发，再在数组中找一对因子，构成它的左孩子的左孩子和右孩子…一直如此构造下去，构造的每一层都可以算是一个新的树。

递归

​ 看构造的描述可以发现，这道题目可以用递归的思路去解。对于一个数a，让它作为根节点能够构造出来的所有带因子的二叉树为：它本身、它的每一对因子(ci,di) (i=1,2...)，ci能够构造出来的带因子的二叉树的数量和di能够构造出来的带因子的二叉树的数量的乘积之和。

记根节点a，它的因子节点为(ci, di) (i=1,2...)，可得如下公式:
$$
f(a) = \sum_{i=1}^nf(c_i)*f(d_i)+1
$$
以上公式适用于ci != di的情况，如果ci == di，记为mi,我们需要减去重复计算的这一部分。
$$
f(a) = \sum_{i=1}^nf(c_i)*f(d_i)+1 - \sum_{i=1}^km(i)^2
$$
下面来看看代码思路，定义dfs函数，初始化sum_为一（它本身），遍历数组中的每一个值val，判断该值能否整除根，并且整除下来的值target/val是否存在于数组中，因为数组中的每一个值都会遍历到，因此target/val和val相等的情况只会被计算一次，因此直接 sum_ += dfs(s, val, arr) * dfs(s, target/val, arr)即可。

最后返回sum_ % mod，递归的方法用于本题是超时的。

动态规划

​ 本题使用动态规划的方法可以节省时间的开销，首先我们可以将数组排序，因为一个数的因子一定比它本身小，在递归的过程中我们可以发现递归到的某一个值无论递归多少次都是相等的，这就给我们两点启发：

  1. 可以用记忆化搜索的方式，存储已经计算过的值。

  2. 可以使用递归的数组，用空间换时间的方式，自小到大依次计算带因子的二叉树的数量。

  3. 递推的公式和上面类似
    $$
    dp[i] = \sum_{m=1}^kdp[c_m] * dp[d_m] * 2+1 +\sum_{j=1}^gdp[m_j]^2 ,(c_m != d_m)
    $$

学习到了灵神的思路，因为遍历的过程中遍历的是值，又因为需要用下标去减少遍历的次数，可以用字典存储值和下标的对应关系。

1921. 消灭怪物的最大数量

解题思路

​ 读一眼题目就能发现其实这和植物大战僵尸有些类似，怪物一定会在某一分钟到达，因此用dist和speed统一度量，定为怪物到达城市的时间即可，也就是说，拿dist=[1,3,4]和speed=[1,1,1]来说，它们分别会在[1,3,4]时间到达城市，将它们进行排序，如果前后两个时间点不同，那么它们一定可以依次消灭掉，如果前后两个时间点相同，如果它们距离到达城市的时间大于等于2，那么它也可以被依次消灭掉。

​ 每次将数组中的时间减一，这样的操作时间复杂度较高，因此我们定义一个distance，在每次计算中将distance加一，如果时间大于这个distance且前后两个相等的话，就返回目前的怪物数量。

解题思路

​ 为了使得二叉树尽可能高度是平衡的，尽量将数组平均分配为两个部分，一个部分作为左子树，一个部分作为右子树，同理，将左子树再一分为二，一个作为左子树的左子树，另一个作为左子树的右子树…不断这么分下去并且拼接成一棵树，由于是平均分配的，它们的左子树和右子树的高度差绝对值不会超过1。

110. 平衡二叉树

解题思路

​ 这道题目和上面那道题目有一定的类似性，一个是构造平衡二叉树，另一个是判断是否为高度平衡的二叉树。既然是一个逆向的过程，那么该如何进行判断呢？

​ 使用递归的方法，在递归的过程中判断左右子树的高度差是否小于等于1，时间复杂度来到了$O(n^2)$,即遍历一次，每个节点在判断一次高度差。

​ 判断高度差是分别计算左右子树的深度，然后让他们相减得到的值就是它们的高度差了。
